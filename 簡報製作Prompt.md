## 6️⃣ 技術問題與解決（2-3 頁）

### 頁面 6-1：遇到的技術問題
**標題**: 開發過程的挑戰

| # | 問題 | 現象 |
|---|------|------|
| 1 | Session 認證失效 | 登入後點擊功能被踢回登入頁 |
| 2 | 資料庫 Schema 不符 | OperationalError: no such column |
| 3 | 跨平台字體不一致 | Mac 正常、Windows 顯示錯誤 |
| 4 | 圖片上傳安全性 | 可能上傳惡意檔案 |
| 5 | Vercel 部署限制 | 資料庫和檔案無法持久化 |

---

### 頁面 6-2：解決方案
**標題**: 問題分析與修復

**問題 1：Session 認證失效**
```
原因：SECRET_KEY 每次重啟隨機產生，導致 Session Cookie 失效

修復前：
app.config['SECRET_KEY'] = secrets.token_hex(32)  # 每次不同

修復後：
app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY', 'fixed-key')
```

**問題 2：資料庫 Schema 不符**
```
原因：舊資料庫沒有新增的 user_id 欄位

解決：刪除舊資料庫 → 重新執行 db.create_all()
```

**問題 3：跨平台字體不一致**
```
原因：Cormorant Garamond, Bebas Neue 在 Windows 渲染不佳

解決：改用 Google Noto 字體系列
- Noto Serif TC（思源宋體）
- Noto Sans TC（思源黑體）
```

**問題 4：圖片上傳安全性**
```python
# 限制允許的檔案格式
ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif', 'webp'}

# 安全處理檔名
filename = secure_filename(image_file.filename)
unique_filename = f"{secrets.token_hex(8)}_{filename}"
```

**問題 5：Vercel 部署限制**
```
挑戰：Serverless 環境不支援 SQLite 持久化

建議方案：
- 改用 Railway 或 Render 平台
- 搭配雲端資料庫（Supabase / PlanetScale）
- 圖片存儲改用 Cloudinary
```
